Model Summary

TVMS Phase 1 is a stable taxonomy for team styles, not a fresh clustering exercise every time a user changes season. The public-facing goal is comparability: the same identity should mean the same thing across leagues and across time unless a future model version replaces the taxonomy.

Reference Window

The taxonomy is anchored to a pooled `2020-2023` reference cohort across the supported leagues. That pooled window is used to set the feature scaling and the identity geometry. Teams from `2024+` are then scored against the frozen model rather than re-clustered inside their own season.

Step 1 Load pooled team-seasons from the reference window.
Step 2 Fit the scaler and identity model on that fixed cohort.
Step 3 Freeze the artifact with identity definitions and thresholds.
Step 4 Score `2024+` teams against the same artifact.

Feature Schema

The model uses a fixed nine-feature tactical profile. Those features include value-added passing, receiving, disruption, dribbling, claiming, possession volume, possession quality, attacking directness, and opponent directness. The site should describe those actual inputs rather than a broader or older metric story.

Identity Discovery And Frozen Scoring

Identity discovery belongs to the training window only. Once the taxonomy has been chosen, the site operates as a scoring layer. Each team is transformed by the frozen scaler, evaluated against every identity, and assigned a full score distribution rather than a single opaque label.

  • Discovery is reference-window-only.
  • The exported artifact stores the identity definitions, scaler, and thresholds.
  • The frontend reads the scored outputs from generated JSON assets rather than recreating any model logic in-browser.

Primary, Secondary, And Hybrid Logic

The output is not just “one team, one label.” Every team gets a primary identity, a secondary identity, and a hybrid flag when the top two fits are meaningfully close. That lets borderline teams remain interpretable without pretending the model is maximally certain when it is not.

Primary identity

The best-scoring frozen identity for the team.

Secondary identity

The strongest runner-up profile retained for context.

Hybrid flag

Raised when the top-two score gap is small enough to reflect a meaningful split.

Statistical Explanation Layer

Each assignment comes with a feature-level explanation. The scorer records the winning identity, the runner-up, the score gap, and the highest-separation features that pushed the team closer to the winner. The explorer surfaces that logic so users can read why a team landed where it did.

What the explanation shows
  • Winner and runner-up identity
  • Relative score gap
  • Top separating features
  • Trait z-scores relative to the frozen reference frame

Validation And Limits

The current taxonomy has been reviewed for coherence and naming quality, but not every identity is equally dense in current MLS usage. Sparse identities remain part of the frozen artifact when they are stable cross-league categories, but they should be treated as watchlist identities in future model versions.

  • `Transition Surge` is retained in the artifact but remains sparse in current MLS scoring outputs.
  • `Aerial Progression` is coherent enough to keep, but still lightly represented in MLS.
  • The site is intentionally explicit about these limits rather than smoothing them over with generic confidence language.
