# TVMS Phase 2: Player Archetyping

**Goal:** Classify every player (min 500 mins) into a tactical archetype using their statistical fingerprint. Archetypes describe *how* a player operates, not just *how well*.

**Architecture:** PlayerLoader -> PlayerPreprocessor -> Clustering (reused) -> PlayerArchetype

---

## Feature Set

All g+ values come from `goals_added_above_avg` (already position-normalized by ASA).
Volume stats are normalized **per 96 minutes** to allow cross-league/position comparison.

| Feature | Source | Rationale |
|---|---|---|
| `g_passing` | `get_player_goals_added` | Distributes ball efficiently |
| `g_receiving` | `get_player_goals_added` | Finds pockets, holds up play |
| `g_shooting` | `get_player_goals_added` | Creates/converts chances |
| `g_dribbling` | `get_player_goals_added` | Carries ball in transition |
| `g_interrupting` | `get_player_goals_added` | Defensive workrate |
| `xgoals_p96` | `get_player_xgoals` | Shot quality volume |
| `xassists_p96` | `get_player_xgoals` | Chance creation volume |
| `passes_oe_p100` | `get_player_xpass` | Pass quality above model |
| `avg_vertical_distance` | `get_player_xpass` | Directness of play |
| `share_team_touches` | `get_player_xpass` | Involvement in build-up |

**Fouling is excluded** — it measures discipline, not tactical identity.

---

## Method

1. Filter: `minutes_played >= 500`
2. Scale: StandardScaler across all 10 features
3. Reduce: PCA to explain 90% variance (for visualization only)
4. Cluster: GMM + BIC (same engine as Phase 1)
5. Label: Centroid scan → human-readable archetype name

---

## Archetype Taxonomy (Expected)

| Archetype | Defining Features |
|---|---|
| The Finishers | High `g_shooting`, `xgoals_p96` |
| The Creators | High `g_passing`, `xassists_p96`, `passes_oe_p100` |
| Progressive Carriers | High `g_dribbling`, `avg_vertical_distance` |
| Box Infiltrators | High `g_receiving`, `xgoals_p96` |
| The Enforcers | High `g_interrupting` |
| The Metronomes | High `share_team_touches`, `passes_oe_p100` |
| Balanced Roles | No extreme outlier features |

## Output Files (per league/season)

- `assets/players_{league}_{season}.json` — per-player archetype + feature z-scores
- `assets/player_metadata_{league}_{season}.json` — model stats, BIC winner
